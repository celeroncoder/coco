# Changelog Template

All notable changes to this project will be documented in this file.
This project adheres to [Semantic Versioning](https://semver.org/).

## Naming Convention

Use the following naming convention for new changelog files:

- **Format**: `NNN-feature-name.md` (e.g., `001-document-upload-proxy.md`, `002-auth-refactor.md`)
- **Numbering**: Sequential, zero-padded (001, 002, 003, etc.)
- **Name**: Descriptive kebab-case name of the feature/change

**How to use this template:**

1. Create a new file: `NNN-feature-name.md` (replace NNN with next sequential number)
2. Start with an "Unreleased" section.
3. Group changes by type for clarity.
4. Document breaking changes and deprecations clearly.
5. Keep entries human-readable and actionable.
6. Include a summary section explaining the overall purpose and files modified.
7. For more info, see [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).
